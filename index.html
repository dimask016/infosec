💡 Analisa Ancaman, Risiko & Solusi
🔒 Identifikasi Ancaman & Risiko:
  • Cookie tidak aman: `NEXT_LOCALE` tanpa `Secure` dan `HttpOnly` → berpotensi session hijacking.
  • Cloudflare WAF: Bisa memblokir security scanning, namun juga bisa menjadi single point of failure.
  • DNSSEC tidak aktif: Rentan DNS cache poisoning / spoofing.
  • Wildcard SSL Certificate (*.antam.com): Jika private key bocor, semua subdomain terancam.
  • Subdomain dengan service tidak standar: `ictcare.antam.com` menggunakan ZGS (400 Bad Request), bisa jadi vulnerable.
  • IP asli tersembunyi di balik Cloudflare: Bisa memberi rasa aman palsu jika origin server terekspos.
🔧 Solusi Mitigasi:
  • Tambahkan flag `Secure; HttpOnly; SameSite=Lax` pada cookie `NEXT_LOCALE`.
  • Aktifkan DNSSEC pada registrar.
  • Gunakan `Authenticated Origin Pulls` di Cloudflare dan batasi akses origin hanya dari IP Cloudflare.
  • Rotasi wildcard certificate secara berkala dan simpan private key dengan aman.
  • Lakukan security assessment pada subdomain `ictcare.antam.com` dan service Zoho.
  • Terapkan `Content-Security-Policy` yang ketat untuk mencegah XSS.

DDL (Data Definition Language) : Primary Key, Foreign Key, Constraints
-- Tabel domain (parent)
CREATE TABLE domain (
    id SERIAL PRIMARY KEY,
    domain_name VARCHAR(255) NOT NULL UNIQUE,
    registrar VARCHAR(255),
    creation_date DATE,
    expiry_date DATE,
    dnssec BOOLEAN DEFAULT false
);

-- Tabel subdomain (child, FK ke domain)
CREATE TABLE subdomain (
    id SERIAL PRIMARY KEY,
    domain_id INT REFERENCES domain(id) ON DELETE CASCADE,
    host VARCHAR(255) NOT NULL,
    ip_address INET,
    asn VARCHAR(20),
    banner TEXT,
    UNIQUE(domain_id, host)
);

-- Tabel scan_result (child, FK ke domain)
CREATE TABLE scan_result (
    id SERIAL PRIMARY KEY,
    domain_id INT REFERENCES domain(id),
    tool VARCHAR(50),
    finding TEXT,
    severity VARCHAR(20),
    scan_date TIMESTAMP DEFAULT NOW()
);
